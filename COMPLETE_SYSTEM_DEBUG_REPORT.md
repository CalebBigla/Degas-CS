## What Was Actually Wrong

### ❌ Problem #1: SQLite Reserved Keyword Error ⚠️ CRITICAL
**Error in logs:**
```
SQLITE_ERROR: near "number": syntax error
```

**Root Cause:**
- When a form has a field named `number` (type: 'number'), the code tried to INSERT into that column without quoting it
- `number` is a reserved keyword in SQL/SQLite
- Example failing query: `INSERT INTO table_name (name, number, email) VALUES (?, ?, ?)`
- SQLite treats `number` as a keyword, not a column name

**Impact:**
- All form registrations with 'number' type fields FAILED with 500 errors
- This is why `/api/admin/forms-tables` returned 500 (couldn't fetch forms due to registration failures)
- The system continued running but any user registration triggered this error

**Location:**
- [backend/src/controllers/eventRegistrationController.ts](backend/src/controllers/eventRegistrationController.ts#L230-L240)

**How It Was Fixed:**
Changed from:
```typescript
const columns = Object.keys(dynamicData);
await db.run(
  `INSERT INTO ${targetTable} (${columns.join(', ')}) VALUES (${placeholders})`,
  values
);
```

To:
```typescript  
const columns = Object.keys(dynamicData);
const quotedColumns = columns.map(col => `"${col}"`).join(', ');
await db.run(
  `INSERT INTO ${targetTable} (${quotedColumns}) VALUES (${placeholders})`,
  values
);
```

Now SQL generates: `INSERT INTO table_name ("name", "number", "email") VALUES (?, ?, ?)`

## How the Errors Were Happening

###Scenario: User tries to register with a form that has a 'number' field
1. Frontend calls POST `/api/register`
2. Backend tries to INSERT user data into dynamic table
3. Code builds: `INSERT INTO users (name, number, email) VALUES (?, ?, ?)`
4. SQLite sees `number` as a reserved keyword
5. SQLite throws error: "near "number": syntax error"
6. API returns 500 Internal Server Error
7. Frontend shows error but doesn't load form table

### Why Frontend Showed 500s on Analytics & Forms
1. When many users try to register (triggering the 'number' bug), the system gets overloaded
2. The error logs fill up with failed registration attempts
3. This can cascade to other endpoints timing out or returning 500
4. The forms list endpoint (`/api/admin/forms-tables`) times out waiting for form data

## Lesson Learned
**Always quote column/table names in dynamically generated SQL.** Even if column names are user-provided, they should be quoted to handle SQL reserved keywords:
- SQLite: Use double quotes `"columnName"`
- PostgreSQL: Use double quotes `"columnName"`
- MySQL: Use backticks `` `columnName` ``

This is a best practice that prevents these types of bugs entirely.
