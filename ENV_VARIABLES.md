## Backend Environment Variables

Location: `backend/.env`

### Critical Required Variables (App crashes without these)

| Variable | Value | Description |
|----------|-------|-------------|
| `NODE_ENV` | `development` \| `production` | Runtime environment |
| `PORT` | `3001` (local) / `10000` (Render) | Server port |
| `DATABASE_TYPE` | `sqlite` \| `postgresql` | Database engine |
| `JWT_SECRET` | strong-random-string | Auth token signing key |
| `QR_SECRET` | strong-random-string | QR code signing key |

### Strongly Recommended Production Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CORE_USER_JWT_SECRET` | varies | Core user token signing key |
| `CORE_USER_JWT_EXPIRES_IN` | `7d` | Token expiration time |
| `FRONTEND_URL` | none | Frontend domain for CORS |

### Database Configuration

#### SQLite (Local + Render Persistent Disk)
```env
DATABASE_TYPE=sqlite
DATABASE_PATH=./data/degas.db
```

#### PostgreSQL (Cloud Providers)
```env
DATABASE_TYPE=postgresql
DATABASE_URL=postgresql://user:pass@host:5432/dbname
```

Cloud DB providers:
- **Neon** (recommended): https://neon.tech
- **AWS RDS**: https://aws.amazon.com/rds/
- **Google Cloud SQL**: https://cloud.google.com/sql/

### File Upload Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `UPLOAD_DIR` | `uploads` | Upload directory path |
| `MAX_FILE_SIZE` | `10485760` | Max file size in bytes (10 MB) |

### Optional: Cloudinary (Persistent Image Storage)

For images that need to survive redeploys:
```env
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
```

Sign up: https://cloudinary.com (Free tier: 25 GB storage)

### Rate Limiting

| Variable | Default | Description |
|----------|---------|-------------|
| `RATE_LIMIT_WINDOW_MS` | `900000` | Window in ms (15 min) |
| `RATE_LIMIT_MAX_REQUESTS` | `100` | Max requests per IP |

### Optional: Email (Resend)

For sending email notifications:
```env
RESEND_API_KEY=re_your_api_key
EMAIL_FROM=noreply@yourdomain.com
```

Sign up: https://resend.com (Free tier: 3,000/month, 100/day)

### Development Only

| Variable | Default | Description |
|----------|---------|-------------|
| `DEV_MOCK` | `false` | Use fake data (dev only!) |

ŌÜĀ’ĖÅ **NEVER set `DEV_MOCK=true` in production**

## Setting Environment Variables on Render

1. Go to your **Render Dashboard**
2. Select your **Backend Service**
3. Click **Settings** ŌåÆ **Environment** (scroll down)
4. Add each variable in the form:
   - **Key:** `VARIABLE_NAME`
   - **Value:** your value
5. Click **Add** and **Save**
6. Service will automatically redeploy

**Pro Tip:** Use Render's "Generate Value" buttons for secrets:
- Click `GENERATE` next to `JWT_SECRET`, `QR_SECRET`, `CORE_USER_JWT_SECRET`
- Render creates cryptographically secure random values

## Generating Secure Secrets

For strong random secrets, use:

**On macOS/Linux:**
```bash
openssl rand -base64 32
```

**In PowerShell (Windows):**
```powershell
[Convert]::ToBase64String((1..32 | ForEach-Object {Get-Random -Maximum 256} | ForEach-Object {[byte]$_}))
```

**Online (HTTPS only):**
https://www.random.org/bytes/

## Troubleshooting

### App won't start: "Missing JWT_SECRET"
Ō£ģ Solution: Add `JWT_SECRET` to backend `.env` or Render environment variables

### "Cannot connect to database"
Ō£ģ Check that `DATABASE_TYPE` matches your setup (sqlite vs postgresql)
Ō£ģ For PostgreSQL, verify `DATABASE_URL` format: `postgresql://user:pass@host:5432/db`
Ō£ģ Test database connection locally before deploying

### Frontend can't reach API
Ō£ģ Verify `VITE_API_URL` matches backend domain
Ō£ģ Check CORS: backend `FRONTEND_URL` must match actual frontend domain
Ō£ģ Inspect browser console for API errors

### File uploads disappear after redeploy
Ō£ģ Use Cloudinary: Set `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`
Ō£ģ Alternatively, use Render persistent disk (already configured in render.yaml)

## Security Best Practices

1. **Never commit real secrets to GitHub**
   - Only commit `.env.example` files
   - Add `.env` to `.gitignore` Ō£ģ (already done)

2. **Use Render's secret generation**
   - Click "Generate Value" for JWT/QR secrets
   - Never reuse secrets across environments

3. **Rotate secrets periodically**
   - Update JWT_SECRET, QR_SECRET every 6-12 months
   - Invalidate old tokens after rotation

4. **Use HTTPS URLs only**
   - `FRONTEND_URL` should always be HTTPS in production
   - API URLs should be HTTPS

5. **Separate credentials per environment**
   - Don't use same database URL for dev & prod
   - Don't use same Cloudinary account for dev & prod
   - Use different JWT secret for each environment
