## Troubleshooting

### Error: "DATABASE_URL is not set"
**Solution**: 
1. Go to Render Dashboard → Backend service → Environment
2. Add `DATABASE_URL` with your Neon connection string
3. Format: `postgresql://user:password@host.neon.tech/database?sslmode=require`

### Error: "core_users table does not exist"
**Solution**:
1. Restart the backend service to trigger database initialization
2. Check backend logs for initialization errors
3. Verify DATABASE_TYPE is set to `postgresql`

### Error: "Connection timeout"
**Solution**:
1. Check Neon database is active (not suspended)
2. Verify connection string is correct
3. Ensure SSL is enabled in connection string

### Error: 500 on /api/form/login
**Cause**: `users` table doesn't exist or form doesn't exist
**Solution**: Run `setup-neon-form.js` script

### Error: 401 on /api/core-auth/login
**Cause**: Admin doesn't exist in database
**Solution**: Run `setup-neon-admin.js` script

## Database Schema Check

To verify all tables exist, run in Render Shell:
```bash
cd backend
node -e "
const { Pool } = require('pg');
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: { rejectUnauthorized: false }
});

pool.query(\`
  SELECT table_name 
  FROM information_schema.tables 
  WHERE table_schema = 'public'
  ORDER BY table_name
\`).then(result => {
  console.log('Tables in database:');
  result.rows.forEach(row => console.log('  -', row.table_name));
  pool.end();
});
"
```

Expected tables:
- admins
- core_users
- forms
- users
- access_logs
- qr_codes
- form_definitions
- form_fields
- attendance_sessions
- attendance_records
- user_data_links

## Manual Database Setup (If Scripts Fail)

If the scripts don't work, you can create the admin manually via SQL:

1. Go to your Neon dashboard: https://console.neon.tech
2. Open SQL Editor
3. Run this SQL:

```sql
-- Create super admin
INSERT INTO core_users (id, email, password, full_name, phone, role, status, created_at, updated_at)
VALUES (
  gen_random_uuid(),
  'admin@degas.com',
  '$2a$10$YourHashedPasswordHere',  -- You'll need to generate this
  'Super Admin',
  '+1234567890',
  'super_admin',
  'active',
  NOW(),
  NOW()
);
```

To generate the password hash, run locally:
```bash
node -e "const bcrypt = require('bcryptjs'); bcrypt.hash('admin123', 10).then(hash => console.log(hash));"
```

## Common Neon-Specific Issues

### Issue: Database suspended
**Symptom**: Connection timeout errors
**Solution**: Neon free tier suspends after inactivity. Visit Neon dashboard to wake it up.

### Issue: Connection pool exhausted
**Symptom**: "sorry, too many clients already"
**Solution**: 
1. Reduce max pool size in backend
2. Upgrade Neon plan for more connections
3. Check for connection leaks in code

### Issue: SSL certificate errors
**Symptom**: "self signed certificate"
**Solution**: Ensure `ssl: { rejectUnauthorized: false }` in connection config
